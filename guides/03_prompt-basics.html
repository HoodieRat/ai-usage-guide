Prompt Basics

Use this page to adopt a simple prompt blueprint you can reuse everywhere. You’ll get copy-paste recipes, fixes for common misses, and a light way to version your best prompts.

What you’ll learn

  • Structure prompts with a tiny blueprint (goal → audience → context → rules → format).
  • Know when to add examples without blowing your token budget.
  • Write quick follow-ups that reliably fix first drafts.
  • Version and reuse prompts with a lightweight “prompt card.”

The prompt blueprint (copy-paste)

Start every request with these five lines. Keep it short (≤ 200–250 words for the whole message).

Goal: <one sentence of what you want>
Audience: <who this is for>
Context: <only the facts or files that matter>
Rules: <2–4 constraints: tone, do/don’t, length>
Format: <bullets, table, JSON, email, etc.>
Expanded version with clarifier 
You are a careful assistant. If something is missing, ask one clarifying question before drafting.

Goal: <what>  
Audience: <who>  
Context: <key facts/links; attach files if needed>  
Rules: <2–4 constraints>  
Format: <exact structure you want>

For strict structures, see Response Contracts and Structured Output.

Recipe gallery (ready to use)

Rewrite (clear & friendly email)

Goal: Rewrite this email to be clear, friendly, and action-oriented
Audience: coworker waiting on a status update
Context: keep dates and commitments; preserve any links
Rules: ≤120 words; plain language; keep original meaning
Format: email draft

<paste your rough email here>

Summarize (bullets + takeaway)

Goal: Summarize this text with the 5 most important points and a 2-sentence takeaway
Audience: me
Context: only include claims supported by the text
Rules: no hype; no new facts
Format: 5 bullets + "Takeaway" (2 sentences)

<paste article or notes>

Plan (short checklist with durations)

Goal: Create a 6-step plan to <your goal> with realistic durations
Audience: me
Context: I have <constraints: time, tools, budget>
Rules: steps must be doable today; mention blockers
Format: numbered list with mm:ss or h:mm per step

Table extraction (consistent columns)

Goal: Extract a table from the text with the exact columns below
Audience: me
Context: source text follows after the instructions
Rules: do not invent values; leave blanks as ""
Format: Markdown table with columns: Item | Date | Owner | Status

<paste text or meeting notes>

Basic research check (with sources)

Goal: Provide a short answer with 2–3 credible sources I can open
Audience: non-expert
Context: <your question>
Rules: include source titles; note uncertainty or disagreements
Format: 3 bullets + "Sources" list (title — site)

If sources disagree, say how and why briefly.

When facts matter, pair this with Browsing & Search and Citations.

Follow-ups that fix common misses

  • Too long: “Shorten to 120 words. Keep dates and decisions.”
  • Wrong tone: “Rewrite for a friendly, professional tone. Avoid hype.”
  • Off-topic: “Focus only on <X>. Remove everything else.”
  • Missing structure: “Regenerate using this format only: <paste format>.”
  • Uncertain facts: “List 3 claims that need a source and provide one line on how to verify.”
  • Jargon overload: “Rewrite in plain language for a smart 9th-grader.”
  • Need options: “Give 3 variations, each with a one-line rationale.”
  • Need context check: “Before drafting, restate my goal and constraints in 2 lines.”

When to add examples

If you want… Do this Why
Specific tone or structure Paste one “golden” example (short) Models imitate patterns they see
Consistent format across runs Provide a response contract (schema/table) Reduces drift and review time
Reliability under token pressure Summarize inputs; link or attach only what’s needed Keeps key facts inside the window

Step-by-step

  1. Define the outcome. One sentence that states the result and the audience.
  2. List only critical context. Facts, files, links that truly change the answer.
  3. Lock constraints. 2–4 rules (tone, length, do/don’t).
  4. Describe the output. Give an explicit structure (bullets/table/JSON/outline).
  5. Add one example (optional). A short “golden” sample makes the target obvious.
  6. Send and iterate. Use the follow-ups above to correct drift fast.

Make it reusable: the prompt card

Save good prompts with a tiny header so you can find and update them later.

id: email-rewrite-v1
owner: <name>
last_review: 2025-10-30
purpose: Clear, friendly status update email
golden_examples:
  - examples/email_rewrite_good.md
  - examples/email_rewrite_alt.md
notes: Keep dates and commitments; ≤120 words

Store these in a prompts/ folder. See Prompt Library for organizing and versioning.

Tips & pitfalls

  • Tip: Keep prompts short. Leave room for the answer and follow-ups.
  • Tip: Name the audience. It instantly improves tone and level of detail.
  • Pitfall: Vague “do everything” asks. Split into 2–3 steps.
  • Pitfall: Implied formats. Always spell out the structure you want.

Acceptance checklist

  • ✅ Your prompt includes Goal, Audience, Context, Rules, and Format.
  • ✅ You kept it ≤ ~250 words and included only critical facts.
  • ✅ You can fix common misses with one-line follow-ups.
  • ✅ You saved at least one prompt as a “prompt card” for reuse.
