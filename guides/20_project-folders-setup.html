Project Folders Setup

Give every project a predictable home so humans and AI know where to find briefs, prompts, reviews, and final deliverables—without hunting through chats.

Retention reminder: Label every folder with the retention class from Memory & Privacy. Archive or purge restricted data within your policy timeline.

What you’ll learn

  • How to standardize folder hierarchies so ChatGPT prompts reference consistent paths.
  • How file naming conventions support search, reviews, and handoffs.
  • How to document context so new teammates—or new chats—can onboard fast.

Prerequisites

  • Access to your team’s shared drive or workspace with permission to create folders.
  • Project charter or intake brief defining objectives and stakeholders.
  • Alignment with privacy and retention requirements for stored assets.

Planner → Actor → Reviewer walkthrough

Planner prompt

“Design a folder structure for the Q4 campaign. Include numbered folders, README contents, and naming conventions for prompts, drafts, and approval logs.”

Actor response

The assistant outputs the skeleton, recommended file names, and a checklist for what belongs in each directory, plus notes on access levels.

Reviewer check

Reviewer verifies restricted assets have gated folders, confirms links, and asks the model to generate a quick-start README for 00_orientation.

Step-by-step

  1. Create the skeleton. Build numbered folders: 00_orientation, 10_inputs, 20_prompts, 30_working-drafts, 40_reviews, 50_final, 90_archive.
  2. Seed orientation. In 00_orientation, add README.md, project charter, glossary, contact roster, and a link to Prompt Library plus Sessions & Context.
  3. Define conventions. Publish naming rules (e.g., 2025-10-30_status_v1.md) and tag formats ([F1], [REF]) that match your prompts and handoffs.
  4. Map permissions. Mark which folders are safe for ChatGPT attachments and list redaction steps before sharing assets. See File Upload Best Practices.
  5. Schedule hygiene. Update a weekly changelog, move approved deliverables to 50_final, and archive stale drafts.

Output contract

When asking the assistant to propose folder updates or inventories, require:

  • Folder path — absolute or shared path.
  • Purpose statement — why it exists and who owns it.
  • Retention tag — policy class from your matrix.
  • Next action — create/move/archive + owner + due date.

Folder skeleton script (PowerShell)

$folders = @(
  "00_orientation",
  "10_inputs",
  "20_prompts",
  "30_working-drafts",
  "40_reviews",
  "50_final",

Logging

Maintain a lightweight changelog capturing date, owner, action, affected path, and reason. For regulated work, export a monthly snapshot of the tree and store it alongside approvals in 90_archive.

Metrics that matter

  • Setup lead time: Hours from kick-off to completed skeleton (target < 2 hours).
  • Naming compliance: % of files matching the convention regex (automate weekly checks).
  • Archive hygiene: # of drafts > 30 days still outside 90_archive (drive toward zero).

Tips & pitfalls

  • Tip: Pin the folder skeleton in your knowledge base and link it from intake forms so every new project starts consistent.
  • Tip: Rotate README ownership weekly; the calendar reminder keeps the overview fresh without one person bearing the load.
  • Pitfall: Moving files without updating manifests. Always revise prompt references after every relocation to avoid missing attachments.
  • Pitfall: Mixing restricted and open data. Label retention class on each folder and gate permissions before sharing with the assistant.

Recap

Consistent folders make AI collaboration predictable. Build the skeleton, publish naming rules, map permissions, and keep the README fresh so every teammate—or assistant—can jump in confidently.

Acceptance checklist

  • ✅ Standard folder template created for new projects.
  • ✅ README documents objectives, links, retention rules, and contacts.
  • ✅ Naming conventions and guardrails shared with the team and assistants.
  • ✅ Changelog process in place with weekly hygiene review.
  • ✅ Metrics track setup speed, naming compliance, and archive hygiene.
