Handoff Between Chats

Move work across chats, shifts, or collaborators without dropping context by packaging intent, evidence, and owner expectations every time you switch threads.

30-60-90 rule: Spend 30s freezing the current state, 60s drafting the handoff brief, and 90s confirming the recipient understands. Anything longer means your process needs automation.

What you’ll learn

  • How to structure a reusable handoff brief that survives chat boundaries.
  • Ways to coordinate planner → actor → reviewer workflows across separate conversations.
  • How to log handoffs for compliance and measure their quality over time.

Prerequisites

  • A shared storage location (workspace folder, wiki, or project tracker) for session logs.
  • Permissions to export transcripts or copy message links from the originating chat.
  • Named owners for planner, actor, and reviewer roles (role patterns).

Planner → Actor → Reviewer walkthrough

Planner prompt

“Summarize the current status of the website FAQ rewrite, including completed sections, pending decisions, and files the next writer must read. Format it as a handoff brief with owners.”

Actor response

The assistant assembles a brief with objective, context snapshot, linked drafts, open questions, and the exact task the next writer should complete in the new chat.

Reviewer check

Reviewer scans the brief, verifies links and owners, then asks the assistant for a confirmation message to paste into the receiving chat. Reviewer signs off only after the recipient restates the plan.

For context hygiene and restarts, see Sessions & Context and Memory & Privacy.

Operational playbook

  1. Freeze context. Run a recap that captures objective, latest decisions, blockers, and pending tasks. Copy critical message links. (Use the recap template in Sessions & Context → Templates.)
  2. Assemble the brief. Use the template below to list purpose, deliverables, attachments (label as [F1], [F2]), and explicit owner expectations. See File Upload Best Practices for limits.
  3. Launch the new chat. Paste the brief first, reference attachment IDs, and ask the assistant to acknowledge before you add new instructions.
  4. Confirm comprehension. Require the next owner to respond with a one-paragraph confirmation or checklist. Address gaps immediately.
  5. Log the handoff. Record date, sender, receiver, chat links, and outcomes in your collaboration tracker so you can audit throughput later.

Output contract

Every handoff brief must include:

  • Objective: One sentence stating the project goal and success metric.
  • Current status: Bulleted recap of completed work, decisions, and blockers.
  • Inputs: File names with version tags, message links, or knowledge base references.
  • Next action: Assigned owner, deadline, and definition of done for the receiving chat.
  • Escalation path: Who to contact if the handoff stalls or requirements change.

For stable formats, pair with Response Contracts and Structured Output.

Logging

Capture handoffs in your collaboration tracker with fields for sender, receiver, chat URLs, assets shared, confirmation timestamp, and any follow-up issues. Attach the exported brief or message ID so reviewers can audit the transfer quickly.

Metrics that matter

  • Handoff confirmation rate: % of handoffs acknowledged within 2 hours. Target ≥ 95% during business hours.
  • Rework incidents: Number of times the receiving chat re-did work because of missing context. Track root causes.
  • Brief completeness score: Average checklist completion (objective, status, inputs, next action, escalation). Sample weekly.

Tips & pitfalls

  • Tip: Keep a shared snippet library of handoff briefs by workstream so you can paste and adapt fast.
  • Tip: Ask the recipient to restate the next three steps in their own words before you leave the chat.
  • Pitfall: Linking to stale files. Update the manifest and reattach with version tags every time you hand off.
  • Pitfall: Skipping the confirmation log entry. Record sender, receiver, and timestamp so audits stay painless.
