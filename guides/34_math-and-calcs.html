Math & Calcs

Coax accurate calculations from ChatGPT by setting up guardrails, unit checks, and verification loops.

As of October 30, 2025: Finance and operations teams must show evidence of independent validation when AI assists with quantitative work. Save every check.
Pair this with Structured Output (for schemas), Response Contracts (to pin formats), and Planners, Critics & Validators (add a “math validator” step).

What you’ll learn

  • Structuring quantitative prompts so the assistant shows full working steps.
  • Comparing AI results to rule-of-thumb and independent checks.
  • Logging verification evidence so numbers are audit-ready.

Prerequisites

  • Documented formulas, business rules, or reference calculations used by your team.
  • Approved calculator or spreadsheet for verification (Excel/Sheets) or a simple Python notebook.
  • Unit conversion sheet or source-of-truth for constants (e.g., FX rates, densities).

Planner → Actor → Reviewer walkthrough

Planner prompt

“Design a calculation workflow for forecasting Q1 cloud spend. List required inputs, formulas, unit checks, and validation steps.”

Actor response

The assistant drafts an input table, provides formula breakdowns, and proposes a validation routine comparing AI outputs with spreadsheet calculations.

Reviewer check

Reviewer marks which numbers require human sign-off, adds sensitivity scenarios, and sets thresholds for when AI results must be escalated.

Step-by-step

  1. Frame the calculation. State goal, precision, assumptions, and acceptable error margin.
  2. Collect inputs. Provide structured values with units and sources; freeze inputs in a log.
  3. Run AI computation. Require step-by-step working, unit tracking, and justification for each transformation.
  4. Validate independently. Re-run in a trusted tool or via manual arithmetic; record variance and who checked.
  5. Record outcomes. Save final numbers, validation evidence, and caveats in the calculation log for audits.

Output contract

Every AI-assisted calculation must include:

  • Inputs table: Value, unit, source, timestamp.
  • Working steps: Line-by-line math with explicit formulas.
  • Validation summary: Method used, variance, reviewer initials.
  • Assumptions log: Constraints, approximations, or business rules applied.

Copy & paste assets

Math-safe prompt header

ROLE: Careful calculator. Always show step-by-step math and keep units on every number.
TASK: {{what to compute}}.
CONSTRAINTS:
- Return an inputs table (name, value, unit, source).
- Track units through each operation. If a unit is missing, stop and ask.
- After computing, produce a second, independent check (different method).
- Report variance between methods and flag deviations > {{tolerance}}.
RETURN: Markdown sections: Inputs • Working • Independent Check • Result • Assumptions • Risks.

Validation prompt

Review the calculation below and propose three ways to confirm it.
Inputs: {{table}}
Working: {{steps}}
Deliver:
- Rule-of-thumb estimate (show math)
- Precise recalculation workflow (Excel/Sheets or manual)
- Red-flag checklist (units, rounding, bounds)

Unit conversion checklist

- Confirm base units for every input.
- Convert with a documented factor and cite the source.
- Re-run after conversion; compare magnitude and variance.
- Log conversion factors in assumptions.

Variance gating rule (paste into instructions)

If |ai_result - validation_result| / validation_result > {{tolerance}},
STOP and list likely causes (units, rounding, formula). Do not publish a single number.

Logging

Log the session with an input snapshot, the exact prompts, AI output, validation evidence (file or screenshot), and reviewer sign-off. Store artifacts next to the financial/ops model.

Metrics that matter

  • Validation coverage: % of AI calculations cross-checked independently.
  • Variance rate: Average deviation between AI and validated figures.
  • Escalation frequency: Count of calcs flagged for policy/compliance review.

Tips & pitfalls

  • Tip: Repeat units after every step—any blank unit is a sign to halt and clarify.
  • Tip: Keep high-precision math in the working section, then round explicitly for the final output.
  • Pitfall: Trusting a single method. Always run an independent check or spreadsheet model before sign-off.
  • Pitfall: Asking the model to juggle complex formulas in one go; break them into smaller sub-steps or pseudo-code first.

Recap

Accuracy comes from structure: define inputs, show work, verify independently, and save the evidence so stakeholders trust the numbers.

Acceptance checklist

  • ✅ Inputs, units, and assumptions captured in the log.
  • ✅ Working steps shown; formulas explicit.
  • ✅ Independent validation attached; variance within tolerance or escalated.
  • ✅ Sign-offs recorded before numbers reach decision materials.
