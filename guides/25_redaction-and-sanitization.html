Redaction & Sanitization

Strip or mask sensitive data before you paste or attach anything in ChatGPT. Follow this play to keep compliance, contracts, and customer trust intact while still getting useful help.

Zero-tolerance rule: Customer PII, credentials, health data, and contractual secrets must not enter ChatGPT unless your privacy lead has approved a sanitized approach. When unsure, escalate or rewrite with synthetic data.
ChatGPT-specific reminder: Treat the web chat like an external tool. Prefer sanitized snippets over full raw files. For attachments, use redacted copies only and label them with [F1], [F2] plus a one-line abstract (see File Upload Best Practices).

What you’ll learn

  • How to spot direct identifiers, quasi-identifiers, and business secrets that require masking.
  • Manual and semi-automated redaction flows that fit normal ChatGPT web-chat usage.
  • How to track sanitized copies, map placeholders, and restore originals safely.

Prerequisites

  • Your data classification matrix and retention policy (see Memory & Privacy).
  • Basic tooling: find/replace with regex, spreadsheet filters, or a DLP/redaction add-in.
  • A secure place to store mapping sheets (who can reverse placeholders).

Planner → Actor → Reviewer walkthrough

Planner prompt

“We must sanitize a client success report for ChatGPT. Identify sensitive fields, propose placeholders, and outline review steps. Return a checklist I can follow.”

Actor response

ChatGPT drafts a field inventory (PII, financial, credentials), suggests standardized tokens, and outputs a step-by-step redaction flow.

Reviewer check

Reviewer spot-checks context clues, validates placeholders, and runs the QA prompt (below) to confirm nothing sensitive remains.

Step-by-step

  1. Classify the asset. Tag the doc: public / internal / confidential / restricted. Note contract limits and retention.
  2. Detect sensitive fields. Scan for:
    • Direct identifiers: names, emails, phone numbers, account/ID numbers, exact addresses.
    • Quasi-identifiers: rare job titles in small firms, exact timestamps, unique anecdotes.
    • Secrets: API keys, credentials, unreleased features, pricing formulas.
  3. Replace with placeholders. Use stable tokens (e.g., <CLIENT_ID>, <EMAIL>, <ARR_USD_K>) and record mappings in a secure sheet.
  4. Generalize where possible. Round numbers (e.g., $1,234,567 → $1.23M), shift dates (±7 days), swap brand names for roles (<COMPETITOR_A>).
  5. Scrub metadata. Remove comments, tracked changes, hidden sheets, document properties, embedded files, and EXIF in images.
  6. Create the sanitized copy. Save to a sanitized/ subfolder with version tag (e.g., report_sanitized_v3.pdf).
  7. Attach or paste safely. In ChatGPT:
    • Prefer pasting smaller sanitized snippets over huge files to stay within the token window.
    • If attaching, upload only sanitized files and label them [F1], [F2] with a one-line abstract.
  8. Log the action. Update the privacy log with file, version, reviewer initials, and link to mapping sheet.

Output contract (when ChatGPT helps you redact)

Require the assistant’s reply to include:

  • Field list: Each sensitive element, its type (PII/financial/credential/secret), and a placeholder.
  • Method: Replace, mask, aggregate, or remove entirely—with one-line rationale.
  • Audit note: Where the change is logged and who approved it.
  • Reversal instructions: How a human can restore values using the secure mapping (never share the mapping in chat).

Placeholder legend template

| Placeholder      | Description                                  | Source field | Storage location          | Owner        |
|------------------|----------------------------------------------|--------------|---------------------------|--------------|
| <CLIENT_NAME>   | Primary customer name                        | Column B     | Vault/clients/map.csv     | Privacy team |
| <ARR_USD_K>     | Annual revenue (thousands USD, rounded)      | Column K     | Finance secure drive      | CFO analyst  |
| <EMAIL>         | Customer email (fully removed from content)   | Column D     | n/a (not stored)          | n/a          |
| <PROJECT_CODE> | Internal codename                             | Text         | Vault/projects/map.csv    | PMO          |

QA prompt for sanitized text (paste into ChatGPT)

You are a privacy reviewer. Inspect the content below and list:
1) Any strings that look like real names, emails, addresses, or IDs.
2) Any context clues that could re-identify a person or client.
3) Any mentions of credentials, API keys, or internal secrets.
Return "Ready for sharing" only if none are found; otherwise list exact lines and a fix.

“Adversarial” redaction test (optional)

Try to infer any original values from the sanitized text using context alone.
If you can plausibly infer, mark the sentence and suggest a safer rewrite or further aggregation.

Logging

Record each sanitized asset in a privacy log: file name, version, data class, placeholders used, reviewer initials, approval date/time, and link to the secure mapping file. Store logs in a restricted workspace with backups.

Metrics that matter

  • Redaction turnaround: Hours from request to approved sanitized copy.
  • Escaped-data incidents: Times sensitive data was detected post-sanitization (target: zero).
  • Audit readiness: % of sanitized files with complete logs and mapping references.

Tips & pitfalls

  • Tip: Maintain a custom dictionary of project codes and VIP names so manual reviews catch context clues automation misses.
  • Tip: Watermark sanitized copies “AI SAFE” to prevent teammates from grabbing the wrong version.
  • Pitfall: Losing the mapping sheet. Store it in an encrypted vault with backups and record the exact path in the privacy log.
  • Pitfall: Skipping adversarial checks. Run a “try to re-identify” prompt before sharing anything externally.

Recap

Redaction is a habit: classify, replace, generalize, scrub, review, then log. Share only sanitized snippets or files in ChatGPT, label them clearly, and keep your mapping sheet out of the chat.

Acceptance checklist

  • ✅ Privacy log captures sanitized assets with reviewer sign-off.
  • ✅ QA prompt saved for verifying sanitized copies.
  • ✅ Metrics tracking turnaround, escapes, and audit readiness.
