Context from Files & RAG Basics

Blend local documents with prompts using lightweight retrieval techniques in ChatGPT’s web chat before you invest in a custom RAG pipeline.

Scope guardrail: Only use files cleared by Project Folders and log every chunk you share so audits and deletions stay traceable. For sanitization, see Redaction & Sanitization.

What you’ll learn

  • How to chunk, tag, and summarize source files for copy/paste retrieval in web chat.
  • Prompts that orchestrate planner → actor → reviewer loops across multiple files.
  • Manual citation and logging habits that mimic production RAG systems.

Prerequisites

  • Redacted files or permission to use internal documents.
  • A spreadsheet or notes tool for chunk cataloging.
  • Familiarity with Chunking & Abstracts.

Planner → Actor → Reviewer walkthrough

Planner prompt

"We have four policy PDFs. Create chunk IDs, two-sentence summaries, and a table that shows which customer questions each chunk can answer."

Actor response

The assistant outputs a catalog with CH-001 style IDs, summaries, token estimates, and recommended tags.

Reviewer check

The reviewer spot-checks summaries for accuracy, adds missing citations, and asks the assistant to produce a retrieval prompt template that cites chunk IDs.

Step-by-step

  1. Pre-process files. Convert content to plain text, remove sensitive data, and store the clean version in your project folder.
  2. Chunk intelligently. Break content into 200–400 token slices anchored to headings or semantic boundaries. Assign IDs and capture source location.
  3. Abstract each chunk. Record one or two sentences, keywords, and question types the chunk answers. Log this in your chunk catalog.
  4. Assemble context packs. For each question, select three to five relevant chunks, paste them into the template below, and include explicit citation instructions.
  5. Review outputs. Confirm the assistant cites chunk IDs, verify claims against source text, and update the catalog with any gaps.

Output contract

Whenever the assistant answers using supplied context, require:

  • Citations: Reference chunk IDs ([CH-###]) inline after relevant statements. See also Citations & Attribution.
  • Confidence: Label each answer High, Medium, or Low based on coverage from supplied chunks.
  • Coverage gaps: Explicitly note if the context did not contain the answer.
  • Follow-ups: Suggested additional chunks or sources to ingest.

Logging

Store the chunk catalog in your project folder. Record who created each chunk, last verification date, and linked prompt sessions. For regulated teams, archive assistant outputs with citations alongside the catalog.

Practical limits & tips

  • Chat limits vary by plan. If a thread slows or stalls after many uploads, start a new chat and paste the latest catalog subset plus a recap (see Sessions & Context and Handoff Between Chats).
  • Prefer text over heavy PDFs. Paste chunk text directly when possible to avoid parsing misses.
  • Keep packs lean. 3–5 chunks per question usually outperform giant dumps.

Metrics that matter

  • Chunk freshness: Percentage of chunks verified within the past 60 days.
  • Citation adherence: Rate of responses containing required chunk IDs.
  • Gap resolution time: Average days from logged gap to new chunk creation.

Tips & pitfalls

  • Tip: Add token estimates to your catalog so you know when a context pack risks hitting limits.
  • Tip: Ask the assistant to self-audit every answer—“Show the chunk IDs used”—to keep hallucinations in check.
  • Pitfall: Dumping too many chunks at once. Cap packs at five snippets and split complex questions into sub-conversations.
  • Pitfall: Letting chunks go stale. Review anything older than 60 days and refresh summaries before reuse.

Recap

Manual RAG works when you prep clean chunks, log their metadata, and enforce citation rules. Keep the catalog current and monitor coverage gaps so the assistant stays grounded in your source files.

Acceptance checklist

  • ✅ Chunk catalog created with IDs, summaries, and source references.
  • ✅ Prompt template saved with citation and gap instructions.
  • ✅ Logging workflow captures updates and verification dates.
  • ✅ Metrics dashboard tracks freshness, citation adherence, and gap resolution.
  • ✅ Review cadence established for refreshing source documents.
