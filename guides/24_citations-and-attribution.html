Citations & Attribution

Keep ChatGPT outputs verifiable by enforcing source IDs, confidence scoring, and auditable claim logs for every non-trivial statement.

No source, no ship: Treat drafts without citations as brainstorming only. Do not publish externally until each claim is tied to a verified source or explicitly marked [NO SOURCE].

What you’ll learn

  • Prompt patterns that make ChatGPT cite verifiable sources inside the web chat.
  • How to map citations to your own file IDs, chunk catalog, or trusted links.
  • Lightweight logging so audits can trace each claim back to evidence.

Prerequisites

  • A source registry (IDs for uploaded files, internal pages, or approved public links). See Context from Files and Chunking & Abstracts.
  • Awareness of your policy for live lookups (see Browsing & Search).
  • A reviewer assigned to spot-check citations and confidence before release (see Source Quality).

Planner → Actor → Reviewer walkthrough

Planner prompt

“Draft a sourcing plan for the 2025 trend report. Include required citation style, trusted source list, banned sources, and a claim log format.”

Actor response

The assistant proposes inline bracket citations (e.g., [SRC-001] or [CH-014]), confidence labels, and a claim log table referencing the source registry.

Reviewer check

Reviewer validates that each claim has a resolvable source ID or [NO SOURCE], samples links, and runs a verification prompt to re-check claims prior to handoff.

Step-by-step

  1. Catalog sources. Maintain a registry with IDs, titles, types, and reliability ratings. Include locations (folder paths or canonical URLs).
  2. Prime the chat. Paste the allowed sources and the required citation style at the start of each drafting session.
  3. Demand inline IDs. Require bracketed IDs after factual statements: “Revenue grew 18% [SRC-011].” For your own chunks, use [CH-###].
  4. Score confidence. Ask for High/Medium/Low with 1-sentence rationale per claim.
  5. Validate & log. Run a reviewer pass that checks every claim’s source resolves and record results in a claim log stored next to the deliverable.

Inline citation styles for ChatGPT web chat

  • Files you uploaded: [F1], [F2] (label & summarize per File Uploads).
  • Chunks from catalogs: [CH-001], [CH-022] (from your chunk table).
  • Approved public sources: [SRC-101] mapping to a URL in your registry.
  • Unknown: [NO SOURCE] (forces review or web check before publishing).

If you use ChatGPT’s browsing in a session, still convert links to your registry IDs before shipping, and capture a PDF/screenshot to prevent link rot.

Output contract

For each non-trivial claim, the assistant must provide:

  • Claim ID (CL-01, CL-02...).
  • Claim text (quote the exact sentence).
  • Source reference (registry ID(s) like [SRC-001] or [CH-014]).
  • Confidence (High/Medium/Low + 1-line rationale).
  • Status (“Verified”, “Needs review”, “Unsupported”).

Citation enforcement prompt (paste at start of drafting)

Use only the sources listed below. Cite inline with bracketed IDs (e.g., [SRC-001], [CH-014]).
If a claim has no source, append [NO SOURCE] and add it to the claim log.
For each claim, add a confidence label (High/Medium/Low) with a short rationale.
Allowed sources (subset from our registry):
{{paste relevant rows}}

Reviewer verification prompt

Review the draft below. For each bracketed claim:
1) Confirm the source ID exists in the registry and supports the claim.
2) Flag any claim with [NO SOURCE] or mismatched evidence.
3) Output a table: Claim ID | Verdict (Verified/Needs review/Unsupported) | Notes | Fix suggestion.

Logging

Save the claim log alongside the deliverable (e.g., in 40_reviews or 50_final). Include reviewer initials, timestamps, and where each source lives. For web sources, capture a PDF or screenshot and store it with the log.

Metrics that matter

  • Verification coverage: % of claims marked “Verified” before publication (target 100% for external work).
  • Unsupported claim rate: Count of [NO SOURCE] flags per draft (trend toward zero).
  • Review turnaround: Hours from draft to citation sign-off.

Tips & pitfalls

  • Tip: Paste a sample paragraph with perfect citations at the top of each session so the assistant mirrors the format.
  • Tip: Require reviewers to spot-check one claim per source tier—primary, secondary, web—before sign-off.
  • Pitfall: Leaving registry links stale. Replace broken URLs with captured PDFs/screenshots and update the registry immediately.
  • Pitfall: Accepting vague confidence labels. Define clear criteria and demand a one-line rationale every time.

Recap

Ground ChatGPT’s outputs by cataloging trusted sources, enforcing inline IDs, labeling confidence, and maintaining a claim log. Pair this page with Deep Research, Source Quality, and News & Recency for higher-stakes work.

Acceptance checklist

  • ✅ Source registry maintained with IDs, reliability ratings, and locations.
  • ✅ Citation enforcement prompt saved as a reusable snippet.
  • ✅ Claim log created and stored with each deliverable.
  • ✅ Reviewer verification run and documented.
  • ✅ Metrics tracked for coverage and unsupported claims.
