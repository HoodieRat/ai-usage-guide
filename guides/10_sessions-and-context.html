Sessions & Context

Keep long-running conversations aligned by budgeting tokens, logging facts, and packaging handoffs. Use these patterns to restart cleanly when a thread gets long or “sticky,” without losing momentum.

What you’ll learn

  • Set up a durable session header and cadence for recaps.
  • Know when to restart a thread and how to carry context forward.
  • Maintain a compact context log that doubles as an audit trail.
  • Hand off work across teammates or new chats without replaying history.

Heads-up before you start

Long threads degrade. After many turns, replies may slow, drift, or “lock up.” Open a new chat and transfer context (template below).

Memory can get “sticky.” If enabled, past details may over-influence tone. Use Temporary Chat for blank-slate runs, or restate rules in your first message.

Files have limits. Attach only what matters and link to the rest. For current limits and prep tips, see File Upload Best Practices.

New to the web app flow? See How ChatGPT Works for thread hygiene and context transfer basics.

Step-by-step

  1. Seed the session header. Your first turn should name the objective, audience, success criteria, and authoritative files/links.
  2. Track the budget. Keep prompt+answer under ~75% of the model’s working window. When drafts get cut off or drift, plan a restart.
  3. Log decisions. After notable turns, copy the facts/decisions/open questions/next actions into your context log (template below).
  4. Recap on cadence. Every 8–10 messages, ask for a 120-token recap and paste it into the log as the new “source of truth.”
  5. Prepare handoffs. When changing owners or chats, generate a one-page digest and have the recipient restate the plan.
  6. Archive & relaunch. Close a bloated thread and start a new one with the latest digest + links/files.

Templates (copy-paste)

1) Session header (first message)

Project: <name>
Objective: <one sentence outcome>
Audience & tone: <who this is for; tone>
Source of truth: <files/links you will use>
Constraints: <must/ must-not; format; length>
Success looks like: <acceptance criteria>

2) Recap prompt (every 8–10 turns)

Summarize this thread in ≤120 tokens.
Return JSON with keys:
"facts": [ ... ], "decisions": [ ... ], "open_questions": [ ... ], "next_actions": [ ... ]

3) Context log entry (paste in your doc)

date: 2025-10-30
thread: "Sessions & Context"
facts:
  - ...
decisions:
  - ...
open_questions:
  - ...
next_actions:
  - owner: <name>
    action: <what>
    due: <YYYY-MM-DD>

4) Context Transfer (new chat restart)

You are continuing the "Everyday AI Guide" project in a fresh chat.

What to use:
- Latest recap (below)
- Linked files: <list the exact files you want used now>

Task:
- Produce <deliverable> using the recap + files only.
- Return the standard HTML scaffold and keep internal links relative.

Recap:
<paste the most recent 120-token recap or digest here>

5) Handoff digest (between teammates)

Handoff: <project> → <new owner>
Goal now: <one sentence>
Must-use sources: <files/links>
Key decisions so far: <3–5 bullets>
Open questions: <3 max>
Next 2 actions: <bullets with owners/dates>

When to restart a session

  • Cut-off answers or repeated “Continue?” prompts.
  • Drift or contradictions against your own recap or files.
  • Slow, unstable responses or the model starts forgetting recent facts.
  • Big scope change. New chat + Context Transfer keeps threads crisp.

Use Handoff Between Chats for a deeper restart checklist.

Instrumentation (lightweight)

Signal How to track Action
Token pressure Rough count: words × 1.33 ≈ tokens Over ~75%? Summarize or restart with a digest.
Answer quality PASS / REVISION in your log Two REVISIONs in a row → restart + tighter contract.
Latency spikes Note seconds per reply Restart or split work; attach only critical files.
Context fidelity Compare outputs to recap Mismatch → recap again or re-anchor with session header.

Tips & pitfalls

  • Tip: Label attachments [F1], [F2] and reference them by label in prompts.
  • Tip: Keep the recap under 120 tokens so it fits easily in restarts.
  • Pitfall: Relying on memory alone. Always paste the latest recap when restarting.
  • Pitfall: Dragging the whole history forward. Bring only what the next step needs.
