How ChatGPT Works

The ChatGPT web app looks at your latest message, the recent thread, any files you attached, your Custom Instructions, and—if enabled—Memory. Keep threads small and specific, attach only the files that matter, and reset with a new chat when you change tasks.

What you’ll learn

  • How ChatGPT composes each reply using your message, recent context, files, and long-term settings.
  • When to start fresh, attach files, or hand off context so conversations stay accurate.
  • Habits that keep responses verifiable, from quick recaps to lightweight logging.

The simple mental model

  1. Your message = the main instruction.
  2. Recent chat = short-term context (older turns fade).
  3. Custom Instructions (optional) = quiet defaults for tone, role, and preferences.
  4. Attached files = what it can cite or extract from.
  5. Memory / Temporary Chat (optional) = carry certain facts forward, or start with a blank slate.

New task? Use New chat. Big change in direction? Consider Temporary Chat or turn Memory off for that run.

Quick wins in the web app

One thread, one task. New chat for new topics → sharper answers.

Attach the evidence. Paperclip files that truly drive the answer. Add a short “what to read for” summary.

Recap before answering. Ask: Before you answer, restate my goal and what info you'll use.

Heads-up: long threads & “sticky” memory

  • Long chats can slow down or get inconsistent. If the thread feels “stuck,” start a fresh chat.
  • Memory can over-influence tone or defaults. If a reply feels off because of past runs, use Temporary Chat or ask it to forget a specific memory, then restate today’s goal.
  • Keep projects going with Context Transfer. When you rescope or start a fresh chat, copy a short project recap + links/files so the new thread picks up exactly where you left off. See Handoff Between Chats and Sessions & Context.

Context Transfer (copy-paste)

Project: <name>
Goal right now: <one sentence>
What to use: <files/links> + 3–5 bullets of key facts
Constraints: <audience, tone, format>
Next step for you: <exact deliverable needed now>

Step-by-step: a clean conversation

  1. Start fresh: New chat. First message states goal, audience, and format.
  2. Defaults (optional): Add or adjust Custom Instructions in Settings → Personalization.
  3. Add files when needed: Attach only what you’ll cite. For long docs, include a 3–5 bullet “highlights” preface.
  4. Plan, then draft: “Outline how you’ll solve this in 3–6 steps.” Fix the plan before the draft.
  5. Verify: If facts matter, ask for sources/citations or do a quick check before sharing.

Try it: three tiny exercises

  1. Recap check: “Before answering, restate my goal and constraints in 2 lines.” Fix any mismatch.
  2. Evidence attach: Upload one short file and ask for a 5-bullet summary with a 2-sentence takeaway.
  3. Context transfer: Start a new chat and paste the Context Transfer block above; ask for the next step.

Tips & pitfalls

  • Tip: Ask for a recap before big answers to catch misunderstandings early.
  • Tip: Summarize attached files in 3–5 bullets so they stay in the active context window.
  • Pitfall: Letting Memory drive tone forever. Audit stored memories monthly and prune outdated ones.
  • Pitfall: Endless threads. Start a new chat once outputs drift or feel slow, then paste a Context Transfer.

60-second recap & next steps

Great results come from clean threads, the right files, short specific asks, and a reliable handoff when you reset. Use Memory wisely, Temporary Chat for blank-slate runs, and Context Transfer to keep projects moving.
